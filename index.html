交易匯入流程｜新舊版比對

UI/UX 改版差異說明 — 供 RD 開發參考

上傳明細，差異如下

1. 使用者會先在交易匯入總覽頁，選擇好要上傳的資金帳戶
2. 進入上傳頁，已知要上傳的資金帳戶，因此不需要「選擇資金帳戶」
3. 上傳檔案後，上傳的檔案資訊會在上傳區塊的外面

補充 後續匯入成功後，應自動跳轉到「該資金帳戶的暫存區」，並且自動載入匯入成功的資料 （切版程式碼無法呈現此步驟）

建立單筆交易，差異如下

1. 進入暫存區會先選擇資金帳戶
2. 單筆資料建立成交易的其餘流程相同

補充 建立成功後，應顯示右上角成功提示訊息 （Scribe 工具在改版後流程無法呈現此步驟）

多筆資料批次套用欄位內容，（包含同時選擇收款及付款資料情境），差異如下

批次套用有兩種形式：Bar & 面板
- Bar：快速套用單個欄位內容，依據上方的「交易類型」呈現可套用的單個欄位選項
- 面板：可套用全部欄位內容，但若上方「交易類型」選擇「不限」，會無法套用收付原因

補充 細節請檢閱「批次套用說明：Bar 與面板」
現況 改版前流程 - 同時選擇收款及付款資料批次套用
現況 改版前流程 - 選擇單一交易類型（收款 / 付款）批次套用
優化 改版後流程

批次套用說明：Bar 與面板 詳細流程請看此

將多筆資料套用共同欄位資訊有兩種形式 → 未來希望「批次編輯」頁也採相同形式 /accountings/batches/init

單個欄位 (收合)：Bar

  • Bar 懸浮在資料列表上方，且維持置底，套用資料後就關閉個別 popup 視窗
  • 交易類型 - 不限：對象 / 標籤
  • 交易類型 - 收款：收款原因 / 對象 / 標籤
  • 交易類型 - 付款：付款原因 / 對象 / 標籤
  • 點擊 popup 中「套用」鈕，將填寫的內容，套用到列表中勾選的資料，並關閉 popup

多個欄位 (展開)：面板

  • 面板懸浮在資料列表上方，且維持置底，套用資料後，面板維持展開狀態
  • 交易類型 - 不限：收付原因 (disabled) / 對象 / 標籤 / 附註 / 手續費 / 營業稅 / 憑證日期
  • 交易類型 - 收款 / 付款：收款原因 / 對象 / 標籤 / 附註 / 手續費 / 營業稅 / 憑證日期
  • 每個欄位需先點擊「鉛筆」圖示，讓欄位變成可編輯狀態，才能編輯該欄位 → 才能讓使用者清空欄位中的內容
  • 若沒有可編輯狀態，套用欄位內容時，會無法分辨使用者是刻意要清空欄位內容還是只是不填寫該欄位內容
  • 「套用」鈕：將面板上已填寫內容的欄位，套用到列表中已勾選的資料，相應的欄位內
  • 「清除」鈕：將面板上已填寫內容的欄位都清空，

共用功能

  • 新增「已選擇筆數」功能：點擊後，會取消勾選的資料，並關閉面板 or Bar
  • 「收合」/ 「展開」按鈕：可切換 Bar＆面板型態
